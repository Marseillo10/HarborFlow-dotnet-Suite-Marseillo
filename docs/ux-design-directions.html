HarborFlow Directions

Layouts

1. Sidebar / Dense 2. Sidebar / Spacious 3. Top Nav / Minimal 4. Top Nav / Rich

Direction 1: Sidebar / Dense

Philosophy: Information-rich command center for power users.

Best for: Users who need to see a lot of data at once and value efficiency over whitespace.

Dashboard

A dense grid of charts and key metrics.

Vessel Map

Map with multiple information panels and controls visible.

Direction 2: Sidebar / Spacious

Philosophy: Balanced and calm, providing focus on one task at a time.

Best for: Users who prefer a less cluttered interface and clear visual hierarchy.

Dashboard

More whitespace around key metrics, larger charts.

Service Requests (Kanban)

A clean, spacious Kanban board for managing service requests.

Direction 3: Top Nav / Minimal

Philosophy: Content-first approach, maximizing vertical space.

Best for: Users who primarily focus on a single content area, like the map.

Vessel Map

An almost full-screen map experience with minimal UI chrome.

Direction 4: Top Nav / Rich

Philosophy: A more traditional web app feel with clear separation of sections.

Best for: Users who are familiar with standard enterprise applications.

Service Requests (Table View)

A detailed table view for service requests with rich filtering and sorting options.
